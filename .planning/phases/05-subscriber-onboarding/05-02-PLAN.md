   ```typescript

     SELECT is_admin, full_name, company, title, profile_type, onboarding_completed

   `
   ```

3. Compute whether to show onboarding. The wizard should show when ALL of these are true:
   - `profile?.onboarding_completed !== true` (not yet completed)
   - `!isAdmin` (admins skip onboarding)
   - `profile?.profile_type` is set (scenario already selected -- wizard comes AFTER ScenarioPicker)

   ```typescript
   const showOnboarding = !isAdmin
     && profile?.profile_type != null
     && profile?.onboarding_completed !== true
   ```

4. Pass `showOnboarding` as a new prop to `DashboardLayoutClient`:
   ```typescript
   <DashboardLayoutClient
     ... existing props ...
     showOnboarding={showOnboarding}
   >
   ```

**Step 2: Modify `components/dashboard/layout-client.tsx`**

1. Add `showOnboarding?: boolean` to both the DashboardLayoutClient props and the LayoutInner props. Default to `false`.

2. Import `OnboardingWizard` from `@/components/dashboard/onboarding-wizard`

3. Add `useState` import and local state in `LayoutInner`:
   ```typescript
   const [onboardingDismissed, setOnboardingDismissed] = useState(false)
   ```

4. Render the wizard conditionally inside `LayoutInner`, AFTER the `SidebarShell` content (so dashboard is visible behind the overlay):
   ```typescript
   {showOnboarding && !onboardingDismissed && (
     <OnboardingWizard
       accessTier={accessTier}
       onComplete={() => setOnboardingDismissed(true)}
     />
   )}
   ```

   Place this right before the closing `</SidebarShell>` tag, after the children/FreeUserGuard block.

5. Pass `showOnboarding` through from `DashboardLayoutClient` to `LayoutInner`.

**Why `onboardingDismissed` state:** After the wizard calls the API and closes, the server-side `showOnboarding` prop is stale until a page reload. The local state prevents re-render from showing the wizard again in the same session.
