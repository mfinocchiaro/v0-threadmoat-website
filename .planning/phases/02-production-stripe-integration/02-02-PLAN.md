<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Stripe Dashboard Setup (Live Mode)</name>
  <action>
User must complete the following in the Stripe Dashboard. These steps cannot be automated — they require the Stripe account owner's browser session.

**Pre-requisite:** Verify Stripe account is activated at https://dashboard.stripe.com/account (KYC/business info submitted and approved).

**Step 1: Create Products and Prices (Live Mode)**
Ensure the dashboard toggle is set to "Live mode" (not "Test mode").

1. Create Product: "The Forge" (Annual Subscription)
   - Price: $18,999.00/year (recurring, yearly)
   - Note the Price ID (starts with `price_`)

2. Create Product: "The Red Keep" (Annual Subscription)
   - Price: Set appropriate amount/year (recurring, yearly)
   - Note the Price ID

3. Create Product: "2026 Q1 Market State Report" (One-time)
   - Price: $4,999.00 (one-time)
   - Note the Price ID

**Step 2: Register Webhook Endpoint**
- Go to Developers > Webhooks > Add endpoint
- URL: `https://threadmoat.com/api/webhooks/stripe` (your production domain)
- Select events: `checkout.session.completed`, `customer.subscription.created`, `customer.subscription.updated`, `customer.subscription.deleted`, `invoice.payment_failed`, `invoice.payment_succeeded`
- Note the Signing secret (starts with `whsec_`)

**Step 3: Configure Customer Portal**
- Go to Settings > Billing > Customer Portal
- Enable: "Allow customers to switch plans" (add all subscription Prices)
- Enable: "Allow cancellations" (cancel at end of period)
- Enable: "Allow updating payment methods"
- Enable: "Invoice history"
- Optional: Add ThreadMoat branding/colors

**Step 4: Set Environment Variables in Vercel**
- `STRIPE_SECRET_KEY` = sk_live_... (from Developers > API keys)
- `STRIPE_WEBHOOK_SECRET` = whsec_... (from Step 2)
- `STRIPE_PRICE_FORGE_ANNUAL` = price_... (from Step 1.1)
- `STRIPE_PRICE_RED_KEEP_ANNUAL` = price_... (from Step 1.2)
- `STRIPE_PRICE_REPORT_Q1` = price_... (from Step 1.3)
- `NEXT_PUBLIC_BASE_URL` = https://threadmoat.com (your production domain)

Also update local `.env.local` with test-mode equivalents for local development.
