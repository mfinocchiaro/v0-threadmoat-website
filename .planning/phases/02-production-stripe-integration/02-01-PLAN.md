1. **STRIPE_PRICES constant** — reads Price IDs from env vars. Three entries:
   - `forge_annual`: `process.env.STRIPE_PRICE_FORGE_ANNUAL` (fallback empty string, not `!` assertion — graceful when env not set)
   - `red_keep_annual`: `process.env.STRIPE_PRICE_RED_KEEP_ANNUAL`

2. **getInternalProductId(stripePriceId: string): string** — reverse lookup function. Iterates STRIPE_PRICES entries to find matching value, returns the key (internal ID). Falls back to the raw stripePriceId if no match found (safe default for coupon/manual subscriptions). Map keys to the exact internal product_id values that getAccessTier expects:
   - `forge_annual` key maps to `'forge_annual'`
   - `red_keep_annual` key maps to `'red_keep_annual'`
   - `market_report_q1` key maps to `'market-report-2026-q1'` (note: hyphenated, matching products.ts)

3. **getStripePriceId(internalProductId: string): string | undefined** — forward lookup. Given internal ID like `forge_annual`, returns the Stripe Price ID from env. Returns undefined if no mapping exists (used by checkout to decide whether to use Price ID or skip).

Do NOT use `!` non-null assertions on env vars. Do NOT hardcode any `price_*` Stripe IDs. All values come from process.env.

Add a JSDoc comment at top explaining: "Maps between Stripe Price IDs (from dashboard) and internal product IDs (used by getAccessTier in lib/tiers.ts). All Price IDs are env-var driven so test and live modes use different values."
