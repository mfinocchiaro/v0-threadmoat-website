1. System Overview

ThreadMoat Website is a B2B SaaS platform for engineering software and industrial AI market intelligence. It tracks 600+ startups across the industrial software landscape and presents them through 500+ interactive dashboard visualizations in 6 languages. The platform converts visitors into paying subscribers through a tiered access model — from free limited views to enterprise advisory engagements.

What It Does

  • Market intelligence dashboards — 44+ visualization pages covering market, financial, geographic, network, and advanced analysis
  • Multi-language marketing — full internationalization across English, French, Spanish, Italian, German, and Portuguese
  • AI-powered analysis — Claude Sonnet 4.5 generates narrative assessments (Impressions, Conclusions, Beware, Overlooked Opportunities)
  • Custom report builder — users select companies, toggle sections, include AI narrative, and export PDF/markdown reports
  • Subscription commerce — Stripe-powered checkout with one-time and annual subscription products

# Email
RESEND_API_KEY=re_...                        # Resend transactional email

# Data source
AIRTABLE_TOKEN=pat...                        # Airtable Personal Access Token
AIRTABLE_BASE_ID=app...                      # Airtable base ID

# AI
ANTHROPIC_API_KEY=sk-ant-...                 # Claude API key for AI narratives
Security: Never commit .env.local to git. The .gitignore already excludes it. For production, set all variables in the Vercel dashboard.

Name Masking

Non-subscribers see obfuscated company names via lib/name-masking.ts. This masks company names, URLs, and other identifying details in dashboard views and API responses. Full data is only revealed after Stripe checkout completes and the user's tier is upgraded.

Coupon System

Partners and early adopters can receive coupons via the admin API (/api/admin/coupons). Coupons map to specific tier upgrades and can be one-time or multi-use.

Onboarding

New subscribers are guided through an onboarding wizard that collects their role, company type, and investment focus areas. This data powers the thesis context and personalized dashboard defaults.

Configuration

  • Locale prefix: as-needed — English pages have no prefix (/pricing), other languages use their code (/fr/pricing)
  • Translation files: stored in messages/ as JSON files (one per language)
  • Language switcher: uses flag emojis in the navigation bar
Brand names are never translated. ThreadMoat, Digital Thread, Analyst, and Strategist remain in English across all locales. These terms are used as product/tier names and must be consistent.

Adding a Translation Key

  1. Add the key to messages/en.json
  2. Add translations to all other locale files (fr.json, es.json, it.json, de.json, pt.json)
  3. Use useTranslations('namespace') in the component to access it

9. AI Narrative Engine

The AI narrative endpoint generates analytical assessments of companies using Claude Sonnet 4.5 via the Vercel AI SDK.

Endpoint

POST /api/ai/narrative

Output Sections

SectionPurpose
ImpressionsFirst-look assessment of the company's market position and technology
ConclusionsInvestment-grade conclusions based on financial and competitive data
BewareRisk factors, red flags, and competitive threats to monitor
Overlooked OpportunitiesUnderappreciated strengths, adjacent markets, and strategic angles

The endpoint receives a company's full data profile (from the loaded CSVs) and generates structured narrative sections. It is used by the custom report builder and is gated behind Analyst tier and above.

Cost: Each AI narrative call consumes Claude API tokens via the ANTHROPIC_API_KEY. Monitor usage in the Anthropic dashboard.

10. Custom Report Builder

The report builder allows subscribers to create custom market intelligence reports.

Workflow

  1. Select companies — choose from shortlist or search the full database
  2. Toggle report sections — include/exclude company overview, financials, competitive analysis, investor landscape
  3. Include AI narrative — optionally generate Claude-powered analysis for each selected company
  4. Capture chart screenshots — embed relevant dashboard visualizations in the report
  5. Export — generate PDF or markdown output

Reports are available to Analyst (one report per purchase), Strategist (unlimited), and Advisory tier users.

Webhook Events Handled

  • checkout.session.completed — upgrade user tier after successful payment
  • customer.subscription.updated — handle plan changes
  • customer.subscription.deleted — downgrade user to Recon on cancellation
  • invoice.payment_failed — flag account for follow-up
Critical: The STRIPE_WEBHOOK_SECRET must match the webhook endpoint configured in the Stripe dashboard. A mismatch silently drops all payment events, breaking tier upgrades.

13. Data Update Workflow

To refresh the startup data displayed on the website, follow this step-by-step process:

Step 1: Pull Fresh CSVs from Airtable

# From the threadmoat-website project root
python scripts/fetch_airtable_csv.py

This downloads the latest Airtable views into data/.

Step 2: Copy Heatmap Enrichment Data

# Copy from the Agentic Updates project output
cp ../ThreadMoat-Agentic-Updates/intake_output/heatmap_enrichment.csv data/heatmap_enrichment.csv

Step 3: Commit Changes

git add data/
git commit -m "data: refresh CSVs from Airtable ($(date +%Y-%m-%d))"

Step 4: Deploy

git push origin main

Vercel auto-deploys within ~60 seconds of push.

Frequency: Data updates are typically done weekly, coinciding with the Agentic Updates weekly sweep. The heatmap enrichment CSV is regenerated less frequently (bi-weekly or as needed).

14. Troubleshooting

Common Issues

Build Failures

Run npm run build locally to reproduce the error. Most build failures are TypeScript errors — the error output will show the file, line, and type mismatch. Common causes:

  • Missing props on a component after a refactor
  • Importing a server-only module in a client component ("use client")
  • Unused variables or imports (strict lint rules)

Stripe Webhook Not Firing

Symptoms: users complete checkout but their tier isn't upgraded.

  • Verify STRIPE_WEBHOOK_SECRET matches the secret in Stripe Dashboard → Webhooks
  • Check the webhook endpoint URL is https://threadmoat.com/api/webhooks/stripe
  • Review Stripe Dashboard → Webhooks → Recent deliveries for error codes
  • For local testing, use stripe listen --forward-to localhost:3000/api/webhooks/stripe

CSV Parse Errors

Symptoms: dashboard shows no data or crashes on load.

  • Check CSV files use UTF-8 encoding and comma delimiters
  • Open the CSV in a text editor — look for unescaped commas in text fields or broken line endings
  • Verify column headers match what lib/load-companies-server.ts expects
  • Re-export from Airtable if the file is corrupted

i18n Missing Key Warnings

Symptoms: console warnings like Missing message: namespace.key for locale "fr".

  • Check that the key exists in all 6 files under messages/
  • Keys must match exactly (case-sensitive, same nesting structure)
  • Add the missing translation — next-intl falls back to the default locale (English) but logs a warning

AI Narrative Errors

Symptoms: report builder shows "Failed to generate narrative" error.

  • Verify ANTHROPIC_API_KEY is set and valid
  • Check the Anthropic dashboard for rate limit or billing issues
  • The narrative endpoint has a 30-second timeout — very large company profiles may hit it

Name Masking Showing for Subscribers

Symptoms: paying users still see masked company names.

  • Check that the Stripe webhook processed the payment event (see webhook section)
  • Verify the user's tier in the database: SELECT tier FROM users WHERE email = '...'
  • Clear the user's session (sign out / sign in) to refresh the tier from the database
